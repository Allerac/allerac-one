# Allerac One v0.1.0 Beta Release Plan

## Status

Planned

## Target

`v0.1.0-beta.1` is the first public beta candidate for Allerac One as a platform,
not only as a browser app.

The beta proves three things:

1. the production web app deploys cleanly;
2. the Control API v1 works through the production Cloudflare edge;
3. the first external client class, represented by the Android robot app and Bruno
   smoke flows, can use scoped API keys against production.

## Scope

Included:

- Production deployment from the release commit.
- Control API v1 behind Cloudflare path policy from
  [ADR 0003](../architecture/decisions/0003-expose-control-api-through-cloudflare-path-policy.md).
- Scoped API key smoke for non-browser clients.
- Android robot client as the first real external client.
- Bruno production smoke collection/environment.
- External smoke script or documented manual smoke for `/api/v1`.
- GitHub beta pre-release.

Not included:

- Separate `api` container extraction.
- Headless deployment profile.
- Streaming chat contract.
- Server-to-device channel.
- Device capability registry or `tools:run`.
- Public mobile app store distribution.

## Release Candidate Flow

Use beta tags for this release line:

```text
v0.1.0-beta.1
v0.1.0-beta.2
```

Beta tags target the release candidate commit and should be marked as GitHub
pre-releases.

Promote to a final `v0.1.0` only after the beta has passed production smoke and any
blocking regressions are fixed.

## Pre-Release Checklist

### Code And Docs

- [ ] Merge the Control API Cloudflare edge policy documentation.
- [ ] Confirm `docs/api/openapi/control-api-v1.yaml` matches the implemented
  `/api/v1` routes used by clients.
- [ ] Confirm the Android robot client points to configurable `base_url`, API key,
  provider, and model.
- [ ] Confirm no production API key is committed to the repository.
- [ ] Confirm docs build passes.

### Local Regression

- [ ] Run unit/API contract tests for Control API conversations and API key auth.
- [ ] Run the production build.
- [ ] Run local Bruno smoke against Docker with a scoped API key.
- [ ] Run Android robot client against local Docker once before production switch.

Suggested local commands:

```powershell
npm test -- src/__tests__/api/control-api-conversations.test.ts src/__tests__/services/api-keys/api-key.service.test.ts --runInBand
npm run build
```

### Production Edge

- [ ] Deploy the candidate commit to the Azure VM.
- [ ] Confirm `https://app.allerac.ai` serves the browser UI.
- [ ] Confirm Cloudflare rule for `/api/v1/*`:
  - no cache;
  - no interactive challenge;
  - no Cloudflare Access browser login;
  - passes `Authorization`;
  - allows JSON `POST`, `PATCH`, and `DELETE`;
  - applies API-appropriate rate limiting.
- [ ] Confirm direct VM/IP access is not required by clients.

### Production API Key

- [ ] Create a dedicated API key named `Android Robot Beta Smoke`.
- [ ] Use only required scopes:

```text
profile:read
chat:read
chat:write
```

- [ ] Add `domains:read` only if the smoke includes domain discovery.
- [ ] Store the raw key outside the repo.
- [ ] Verify revocation behavior after smoke using a disposable key.

## Production Smoke

Run the smoke from outside the VM, through Cloudflare.

Required flow:

1. `GET /api/v1/me`
2. `POST /api/v1/conversations`
3. `POST /api/v1/conversations/:id/messages`
4. `GET /api/v1/conversations/:id/messages`
5. Revoke a disposable key and verify `401 unauthorized`

Expected behavior:

- responses are JSON envelopes, not Cloudflare challenge HTML;
- `Authorization: Bearer alr_live_...` reaches the origin;
- wrong or revoked keys fail with Control API JSON errors;
- the synchronous chat response returns an assistant message;
- Cloudflare does not cache conversation responses.

## Bruno Smoke

Create a Bruno environment named `Production`:

| Variable | Value |
|---|---|
| `baseUrl` | `https://app.allerac.ai` |
| `apiKey` | Stored locally, never committed |
| `conversationId` | Set by `Conversations / Create Conversation` |

Minimum requests for the beta:

- `System / Me`
- `Conversations / Create Conversation`
- `Conversations / Send Message`
- `Conversations / List Messages`

The Bruno production environment should be safe to commit only if it contains no
secrets. Secret values must stay local.

## Android Robot Smoke

After production API smoke passes, launch the Android robot app with:

```powershell
.\android-platform-tools\platform-tools\adb.exe shell am start `
  -n com.allerac.robot/.MainActivity `
  -e base_url https://app.allerac.ai `
  -e api_key $secret `
  -e provider github `
  -e model gpt-4o
```

Expected behavior:

- the app connects without `adb reverse`;
- tap starts listening;
- a spoken prompt creates or reuses a Control API conversation;
- the assistant reply is spoken by the device;
- pausing the app does not create a Control API error state.

## GitHub Release

Create the beta as a GitHub pre-release.

Suggested release notes:

```text
Allerac One v0.1.0-beta.1

First beta focused on the platform control plane:

- Control API v1 available for scoped API-key clients
- Cloudflare path policy for /api/v1 production access
- Android robot client as first external client prototype
- Agent worker split completed
- Bruno/API smoke path for production validation

Known limitations:

- chat is synchronous, not streaming
- robot app is manually installed APK
- Control API still runs in the app container
- headless mode is not included
```

## Rollback

If production smoke fails:

- keep the GitHub release marked as pre-release;
- do not promote to final `v0.1.0`;
- restore the previous Azure deployment or redeploy the last known good commit;
- revoke any production smoke keys that may have been exposed during debugging;
- record the failure in this document before cutting the next beta candidate.

## Exit Criteria

The beta is valid when:

- production UI is reachable;
- `/api/v1` smoke passes through Cloudflare with a scoped API key;
- Android robot app can complete one production conversation;
- API key revocation is verified;
- no production secret is committed;
- GitHub pre-release is published with known limitations.
